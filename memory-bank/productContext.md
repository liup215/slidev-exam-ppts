# Product Context: slidev-exam-ppts

## 问题陈述

### AI 生成内容的局限性

传统的 AI 内容生成方式存在以下问题：

1. **过度概括**: AI 倾向于将内容简化到"通用"水平，丢失考试所需的细节
2. **缺乏针对性**: 生成的内容不针对特定考纲的要求
3. **信息密度不足**: 备考需要高密度、结构化的信息，而非概述性内容
4. **更新困难**: 当考纲或教材更新时，难以同步更新生成的内容

### 现有解决方案的不足

| 方案 | 问题 |
|-----|------|
| 直接使用 AI 生成 | 内容过于概括，缺乏考试细节 |
| 手动制作 PPT | 耗时耗力，难以维护 |
| 传统笔记软件 | 缺乏与演示材料的联动 |
| 单一输出格式 | 无法满足课堂和课后两种场景 |

## 解决方案：内容切片模式

### 核心理念

**保留原始教材的完整性和细节**，而非依赖 AI 的"理解"和"总结"。

### 内容切片模式

```
原始教材 → 内容切片 → 结构化存储 → 双输出生成
    ↓           ↓            ↓            ↓
  完整内容    知识单元    统一格式    PPT + 笔记
```

### 切片的优势

1. **信息完整性**: 保留原始教材的所有细节和上下文
2. **可追溯性**: 每个切片都可以追溯到原始来源
3. **可复用性**: 同一内容可用于多种输出格式
4. **可更新性**: 局部更新时只影响相关切片

## 使用场景

### 场景一：课堂演示 (PPT)

- **用户**: 教师
- **需求**: 结构化的教学演示，包含图表、重点标注
- **输出**: Slidev 幻灯片
- **特点**: 
  - 视觉化呈现
  - 渐进式展示
  - 支持交互元素

### 场景二：课后复习 (笔记)

- **用户**: 学生
- **需求**: 详细的学习笔记，便于搜索和复习
- **输出**: VitePress 文档网站
- **特点**:
  - 完整内容展示
  - 侧边栏导航
  - 全文搜索支持
  - 响应式设计

### 场景三：自主学习

- **用户**: 自学生
- **需求**: 先通过 PPT 了解框架，再通过笔记深入学习
- **工作流**: PPT 预览 → 笔记详读 → 交叉参考

## 价值主张

1. **对学生**: 获得与考纲精确匹配的高质量学习材料
2. **对教师**: 大幅减少备课时间，同时保证内容质量
3. **对内容维护者**: 更新一次，多处同步
